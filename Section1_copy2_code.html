Things to Know about the Web - Section 1

The Web

The web is is made of many different components, first of which is us, the user. We type in commands on our computer which get sent over the internet from our web browsers to servers using HTTP. The servers interperet the commands and respond accordingly.

HTML

HTML is the basis for everything on the web and stands for Hyper Text Markup Language. HTML constist of:

Text Content - Everything you see
Markup - What it looks like
References - to other documents like images and videos
Links to other pages - Links are referred to as URLs. Clicking on links, moving deeper into web content is why its called the Web.

HTML Markup

HTML Markup is what the content looks like. Markup consist of Tags and Attributes
defined within an Element.

An Element contains of an opening tag, a closing tag, and content inbetween the two tags.
If a tag has only an opening tag, but no closing tag, it is called a Void Tag.

Attributes are also part of an element, so also contained inside a tag. Attributes will point to a value.

More Things to Know about the Web - Section 2

Structure of a Web Page

HTML - The overall structure of a web page, much like the walls of a room
CSS - How the page is styled - much like the decoration on the wall or the color it is painted.
Javascript - Changing an element of a web page, much like flipping a light switch or using a remote control.

Web Browser and Development Tools

The web browser builds a tree-like structure of the web page. This is referred to as the DOM or Document Object Model.
The browser turns HTML tags into elements that form a Tree.

HTML elements in the structure have an opening tag and a closing tag. The information contained between the tags can be Text, Span, Div, Img, Src, etc.

Each browser has its own set of Development Tools which shows the structure and style of the web page.

Elements displayed in the development tools are not the actual code, but the structure of the web page as the browser interprets it.

Languages

HTML and CSS are both "languages".

HTML Language controls the structure of the page and applies specific syntax and rules to the page. The basic word of the HTML Language is a "tag". Tags make up elements.

CSS Language controls the style of the page. CSS uses specific syntax and rules to determine how the HTML elements will appear on the page.

Adding CSS Style - Section 3

CSS Gives Us Stlye!

CSS or Cascading Style Sheets, define how HTML elements are to be displayed in the browser. CSS attributes can be applied directly into the HTML or from an outside .css file. Using a .css file is the preferred way by developers so the HTML and CSS files can be separate and easier to work with.

CSS Saves Us a Lot of Work!

The main function of using CSS is the ability to change the style of HTML documents in one place. By assigning classes to HTML elements, one can simply change the style of an entire class with one change to the .css class file. The main two reasons this is important is that it:
1. Avoids Repetition - The developer does not need to style individual elements or lines of code taking more time.
2. Avoids Mistakes - By making one change instead of several changes, the developer can avoid the potential for costly errors.
